Card Basic — Verso
Qual a diferença entre git reset e git revert?
reset: apaga commits do histórico (reescreve)
→ Use em commits locais não publicados

revert: cria um NOVO commit que desfaz o anterior
git revert abc1234
→ Use em commits já publicados (safe)
Card Basic — Bloco de código
Como criar uma função em bash?
backup() {
  local origem=$1
  local destino=$2
  cp -r "$origem" "$destino"
  echo "Backup de $origem feito!"
}

backup ~/docs ~/backup/
Card Basic — Com dica
Diferença entre git pull e git fetch?
git fetch — baixa mudanças remotas mas não aplica
git pull — fetch + merge (aplica as mudanças)

💡 git pull --rebase é mais limpo: reaplica seus commits em cima das mudanças remotas
Card Basic — Com aviso
Como desfazer o último commit (mantendo as mudanças)?
git reset --soft HEAD~1 — volta um commit, mantém staging
git reset HEAD~1 — volta um commit, mantém no working dir

⚠️ git reset --hard HEAD~1 APAGA tudo. Cuidado!

Comparação — Antes vs Depois
Antes
Como adicionar todos os arquivos modificados para commit?
git add .
Ou, para selecionar arquivos individuais: git add nome_do_arquivo
Depois
Como adicionar arquivos para commit?
git add arquivo.py — arquivo específico
git add . — tudo no diretório atual
git add -p — interativo (escolhe pedaços)

⚠️ Evite git add . cegamente — revise com git status antes
